¿Qué es RDF?

Resource Description Framework (RDF) es un modelo estándar para el intercambio de datos en la Web, permitiendo que la información sea procesada por máquinas.

Interoperabilidad

Permite que diferentes aplicaciones compartan datos sin importar su lenguaje de programación.

Web Semántica

Añade significado a los datos para que buscadores como Google entiendan el contexto.

Modelo de Grafo

Representa información como una red interconectada de recursos y relaciones.

El Modelo de Tripleta

Toda la información en RDF se descompone en tres partes esenciales:

Sujeto

El recurso o entidad que queremos describir identificado por una URI (ej. "Pedro").

Predicado

Define la relación o propiedad (ej. "es autor de").

Objeto

El valor de la propiedad o el recurso con el que se relaciona (ej. "De esa canción").

Evolución: De RDF 1.1 a 1.2

El estándar ha evolucionado para adaptarse a la Web moderna:

  • RDF1.0:Tuvo su origen entre 1999 y 2004, era muy dependiente de XML. Se creía que debia escribirse todo con etiquetas, lo cual hacia dificil la comprensión. Aca se introduce URI (Identificadores Únicos)para cada cosa.
  • RDF 1.1: Introdujo el formato JSON-LD, permitiendo que los datos semánticos se integren fácilmente en aplicaciones web actuales.
  • RDF 1.2 (Próximamente): Introduce RDF-Star, que permite añadir metadatos a las relaciones (como la confianza o la fecha de un dato), algo vital para la Inteligencia Artificial.
